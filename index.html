Do Coupling Metrics Help Characterize Critical Components in Component-based SPL?
An Empirical Study

Abstract

In component-based software product lines (SPL), each component has to encapsulate features, restrict data access, and be replaceable. For critical components, with multiple features and dependencies, these criteria are fundamental for flexible product configuration. Previous work assume that coupling metrics help characterize critical components, but we lack empirical evidence for that assumption. By characterizing critical components, we could help developers identify components that require careful maintenance and evolution. This paper relies on five well-known coupling metrics to compose a strategy for characterizing critical components in component-based SPLs. Our results suggest reasonable strategy's accuracy but the need for using additional metrics.

Coupling Metrics

Coupling measures how much a SPL component provides functionalities to other components, or depends on functionalities provided by other components. That is, coupling metrics aim at capturing when SPL components are strongly attached to the whole SPL. Thus, a high coupling suggests that the SPL components are strongly attached to other components and, therefore play an essential role in the operation of the product line. On the other hand, a low coupling suggests that the SPL component is not essential to operate the product line. In order to characterize critical components in component-based SPL, we used the following conventional coupling metrics.

Metric Description
Coupling Between Objects (CBO) [1] Number of components coupled to one specific component
Fan-in (FANIN) [2] Number of dependencies from other components to a specific component
Fan-out (FANOUT) [2] Number of dependencies from one components to others
Depth of Inheritance Tree (DIT) [1] Maximum depth of a component in the inheritance tree
Number Of Children (NOC) [1] Number of children components of a component

The first column presents the name of each metric. The second column describes the metric.
